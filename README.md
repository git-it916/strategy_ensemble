# Strategy Ensemble — 암호화폐 선물 자동매매 시스템

바이낸스 USDT-M 무기한 선물, 3배 레버리지, 최대 1 포지션 집중 투자.
10개 알파 시그널 → 가중합 앙상블 → 규칙 기반 의사결정 → 자동 실행.

```
conda activate strategy_ensemble
python scripts/run_daemon.py          # 실전
python scripts/run_daemon.py --dry-run  # 시그널만 로깅
```

## 3. 10개 알파 — 각각의 역할과 효용

### 알파 가중치 구성

```
인트라데이 (78%)                          배경 컨텍스트 (22%)
┌──────────────────────────────────┐    ┌──────────────────────────┐
│ MomentumMultiScale    25% ████▓ │    │ FundingCarryEnhanced  8% │
│ IntradayVWAPV2        20% ████  │    │ MomentumComposite     6% │
│ OrderbookImbalance    18% ███▓  │    │ MeanRevMultiHorizon   5% │
│ IntradayRSIV2         15% ███   │    │ DerivativesSentiment  3% │
└──────────────────────────────────┘    └──────────────────────────┘
                                        SpreadMomentum    0% (비활성)
                                        VolatilityRegime  0% (비활성)
```

### 알파별 상세

#### ① MomentumMultiScale — 25% (핵심 타이밍)

- **데이터**: 5분봉 (6/18/36개 = 30분/90분/180분)
- **계산**: 3개 시간대 가중 수익률 (40%/35%/25%) → 변동성 정규화 → tanh 스케일링
- **효용**: 단기 가격 추세의 강도와 방향을 포착. 30분 모멘텀이 가장 민감, 180분이 추세 확인
- **특징**: 가속도 보너스 (30분 모멘텀 > 90분이면 강화), 심볼별 EMA 스무딩으로 노이즈 감소
- **약점**: 횡보장에서 잦은 방향 전환 → 거짓 신호 발생

#### ② IntradayVWAPV2 — 20% (평균회귀 브레이크)

- **데이터**: 1시간봉 (최근 24개 = 1일)
- **계산**: 24시간 VWAP 대비 Z-score → |Z| > 0.8이면 반대 방향 신호
- **효용**: "가격이 VWAP에서 너무 벗어나면 되돌아온다"는 평균회귀 논리. 모멘텀 알파의 과열 브레이크 역할
- **특징**: 추세 필터 — 4시간 수익률 > 2%이면 신뢰도 감소 (강한 추세에선 평균회귀가 틀림)
- **약점**: 강한 돌파 구간에서 역방향 신호 → 모멘텀과 충돌

#### ③ OrderbookImbalance — 18% (미시구조)

- **데이터**: 실시간 오더북 (3구간: 0-5, 5-10, 10-20 레벨)
- **계산**: 구간별 bid/ask 불균형 가중합 (50%/30%/20%) → 30분 이동평균 → tanh 스케일링
- **효용**: 주문 대기열의 매수/매도 힘 균형을 직접 측정. 다른 알파보다 선행성이 있을 수 있음
- **특징**: EMA 스무딩으로 순간 노이즈 감소
- **약점**: 대형 주문이 취소되거나 스푸핑(의도적 허위 주문)에 취약. 미검증 알파로 가중치 변동 가능

#### ④ IntradayRSIV2 — 15% (과매수/과매도 필터)

- **데이터**: 1시간봉 (14개 = 14시간)
- **계산**: Wilder RSI(14) → RSI < 30이면 매수, RSI > 70이면 매도
- **효용**: 극단적 가격 움직임 후 반전 포착. 모멘텀 알파가 과열 진입하는 것을 막는 안전장치
- **약점**: 강한 추세에서 RSI가 오래 극단에 머무름 → 조기 반전 신호

#### ⑤ FundingCarryEnhanced — 8% (구조적 엣지)

- **데이터**: 펀딩비율 (14일 = 42개 8h 구간) + OI + 일봉
- **계산**: -평균펀딩 × 5000 → 펀딩속도 보정 → 군중 할인 → 추세 필터
- **효용**: 펀딩비 음수 = 숏 쪽이 롱에게 지불 → 롱이 구조적으로 유리. 8시간마다 발생하는 "배당" 같은 수익원
- **특징**: OI 급증 + 펀딩 극단 시 신뢰도 50% 감소 (군중 위험), 추세 역방향이면 신뢰도 40% 감소
- **약점**: 펀딩비 변동이 느려서 (8시간 갱신) 급변하는 시장에 늦게 반응

#### ⑥ MomentumComposite — 6% (중기 방향)

- **데이터**: 일봉 (20일)
- **계산**: 절대모멘텀(40%) + 상대모멘텀(30%) + 위험조정모멘텀(30%) 합산
- **효용**: 20일 추세 방향을 확인. 인트라데이 알파가 중기 추세와 같은 방향인지 배경 확인용
- **약점**: 일봉 기반이라 갱신이 느림. 인트라데이 봇에서는 보조 역할

#### ⑦ MeanReversionMultiHorizon — 5% (장기 과열 경고)

- **데이터**: 일봉 (3/20, 5/60, 10/120일 Z-score 3개)
- **계산**: 3개 시간대 Z-score 평균 → -tanh(z/2) → 볼린저 이탈시 보너스
- **효용**: "120일 기준으로 현재 가격이 비정상적으로 높다/낮다"는 장기 관점 제공
- **약점**: 장기 지표라 단기 매매에서 활용도 제한적

#### ⑧ DerivativesSentiment — 3% (파생상품 심리)

- **데이터**: OI 변화율, 펀딩비, 롱숏비율 (실시간)
- **계산**: 스퀴즈 점수(OI + 펀딩 불일치) × 60% + 롱숏비율 조정 × 40%
- **효용**: OI 급증 + 펀딩 역방향 = 스퀴즈 위험 감지. 군중 포지션 과밀 경고
- **약점**: 낮은 가중치(3%). 단독으로 진입 방향을 바꿀 만큼 강하지 않음

#### ⑨ SpreadMomentum — 0% (비활성)

- 오더북 스프레드 변화 × 가격 방향. 미검증으로 비활성화

#### ⑩ VolatilityRegime — 0% (비활성)

- 시장 변동성 레짐 판단 (방향 신호 없음, 신뢰도만 출력). 비활성화

### 카테고리별 효용 정리

| 카테고리 | 알파 | 합산 가중치 | 역할 |
|---------|------|-----------|------|
| **모멘텀** | MomentumMultiScale, MomentumComposite | 31% | 진입 방향과 타이밍의 핵심 |
| **평균회귀** | IntradayVWAP, IntradayRSI, MeanRevMulti | 40% | 과열 브레이크 + 반전 포착 |
| **캐리** | FundingCarry, DerivativesSentiment | 11% | 구조적 방향 + 스퀴즈 경고 |
| **미시구조** | OrderbookImbalance | 18% | 주문 흐름 선행 신호 |

**설계 의도**: 모멘텀이 방향을 제시하고, 평균회귀가 과열을 억제하고, 미시구조가 타이밍을 보완하는 3중 구조. 모멘텀(31%) < 평균회귀(40%)로 "추세 추종하되 과열에선 멈추는" 보수적 성향.

## 4. 앙상블 — 시그널을 하나로 합치는 방법

### 계산 공식

```
                    Σ (score_i × confidence_i × weight_i)
ensemble_score = ─────────────────────────────────────── × vol_confidence
                          Σ weight_i
```

- **score**: 알파가 출력한 방향 신호 (-1 ~ +1)
- **confidence**: 알파가 자체 판단한 신뢰도 (0 ~ 1). 데이터 부족, 추세 충돌 시 감소
- **weight**: settings.py의 ALPHA_WEIGHTS
- **vol_confidence**: VolatilityRegime이 출력한 시장 변동성 신뢰도 (최소 0.5로 캡)

### 특징

- **confidence 가중**: 단순 score × weight가 아니라, 알파가 스스로 "이 신호에 확신이 없다"고 판단하면 자동으로 기여도가 줄어듦
- **NaN 방어**: score나 confidence가 NaN이면 해당 알파 스킵
- **vol_confidence 캡**: 최소 0.5 → 단일 알파(VolatilityRegime)가 전체 시그널을 0으로 만드는 것 방지
- **Stacking 메타모델**: 코드 존재하지만 현재 비활성. 60일 이상 데이터 축적 후 자동 전환 예정

### 스코어 해석

| ensemble_score | 해석 |
|---------------|------|
| +0.14 이상 | 롱 진입 후보 |
| -0.16 이하 | 숏 진입 후보 |
| ±0.04 미만 | 시그널 거의 없음 (WEAK) |
| ±0.08 미만 | 시그널 약화 중 (FADE) |

## 5. 진입 로직 — 언제 포지션을 여는가

### 임계값 (비대칭)

| 방향 | 임계값 | 근거 |
|------|--------|------|
| **LONG** | score >= **+0.14** | 백테스트 기준 승률 50%+ 유지하는 최저선 |
| **SHORT** | score <= **-0.16** | 숏은 롱보다 엄격하게 (하락장 리스크 비대칭) |

### 진입 프로세스

```
사이클 1: score +0.18 → PENDING (1/2 사이클)
사이클 2: score +0.19 → 상승 확인 (delta >= 0.01) → OPEN!
```

1. **2사이클 연속 확인**: 1번 넘었다고 바로 진입하지 않음. 2사이클(10분) 연속으로 임계값을 넘어야 함
2. **스코어 상승 요구**: 마지막 스코어 - 첫 스코어 >= 0.01. 시그널이 강해지는 추세여야 진입

### 진입 필터 (차단 조건)

| 필터 | 조건 | 이유 |
|------|------|------|
| **과열 방지** | 해당 코인 1시간 수익률 > ±3% | 이미 많이 움직인 후 추격 매수/매도 방지 |
| **BTC 방향** | BTC 1시간 -1% 이하인데 롱 / +1% 이상인데 숏 | BTC 역방향 진입은 대부분 실패 |
| **거래량 확인** | 최근 30분 거래량 / 평균 < 0.7 | 거래량 없는 가격 변동은 신뢰 불가 |
| **쿨다운** | 같은 코인 청산 후 20분 | 연속 손절 방지 |
| **연속 손실 차단** | 같은 코인 3회 연속 손실 → 4시간 차단 | 특정 코인에 대한 시그널 반복 실패 시 차단 |
| **일일 횟수** | 하루 최대 6회 거래 | 과도한 매매 방지 |

### 포지션 사이징

- 잔고 × 95% × 3배 레버리지 = 실질 포지션 크기
- 예: 잔고 $1,000 → 포지션 $2,850

## 6. 청산 로직 — 언제 포지션을 닫는가

### 청산 우선순위 (위에서 아래로)

```
우선순위 1: SL/TP 모니터 (5초 체크, 별도 태스크)
    ├── 하드 SL 도달 → 즉시 청산
    ├── 트레일링 활성 중 하락 → 즉시 청산
    └── 하드 TP 도달 (트레일링 미활성시만) → 즉시 청산

우선순위 2: 규칙 엔진 (5분 체크, 메인 루프)
    ├── 최소 보유 15분 미만 → 청산 불가
    ├── REVERSAL: 반대 방향 진입 임계값 도달 → 즉시 청산
    ├── WEAK: |score| < 0.04가 10분 지속 → 청산
    ├── FADE: |score| < 0.08이 15분 지속 → 청산
    └── SWITCH: 30분 이상 보유 + 더 강한 코인 발견 → 청산 후 즉시 교체
```

### 규칙 엔진 청산 상세

#### REVERSAL (시그널 반전)

```
롱 보유 중: 현재 score < -0.16 (숏 임계값) → 즉시 청산
숏 보유 중: 현재 score > +0.14 (롱 임계값) → 즉시 청산
```

시장 방향이 완전히 뒤집혔다는 뜻이므로 최소 보유 시간(15분)만 지나면 즉시 실행.

#### WEAK (시그널 소멸)

```
롱 보유 중: effective_score < 0.04 → 타이머 시작 → 10분 지속 → 청산
(effective_score = score if LONG, -score if SHORT)
```

시그널이 거의 사라진 상태. 10분간 회복하지 않으면 "시그널 없이 포지션 들고 있는 것"이므로 청산.

#### FADE (시그널 약화)

```
롱 보유 중: effective_score < 0.08 → 타이머 시작 → 15분 지속 → 청산
```

시그널이 진입 임계값(0.14)의 ~57% 수준으로 약해진 상태. WEAK보다 느슨하지만 더 오래(15분) 기다림. 타이머 중 스코어가 회복되면 리셋.

#### SWITCH (코인 교체)

```
조건 (모두 충족):
  - 보유 30분 이상
  - 현재 코인의 effective_score < 0.08 (FADE 수준)
  - 다른 코인의 |score|가 진입 임계값 이상
  - 같은 방향이면 스코어 차이 >= 0.10
  - 반대 방향이면 현재 스코어가 진입 임계값 - 0.08 이하
```

현재 코인보다 훨씬 강한 시그널이 나온 코인이 있을 때, 청산 → 즉시 새 코인 진입.

## 7. 익절 로직 — SL/TP/트레일링 상세

### SL/TP 설정 (비대칭)

| 항목 | LONG | SHORT | 3x ROI 기준 |
|------|------|-------|------------|
| **Stop Loss** | -3.5% | -2.5% | 롱 -10.5%, 숏 -7.5% |
| **Take Profit** | +10% | +8% | 롱 +30%, 숏 +24% |
| **RR 비율** | 2.86:1 | 3.2:1 | — |

- 숏의 SL이 더 타이트한 이유: 하방보다 상방 돌파가 더 빠르고 폭발적이므로 빠르게 손절
- SL/TP 가격은 진입 시점에 계산되어 `PositionStore`에 저장 (봇 재시작해도 유지)

### 트레일링 스탑

```
수익 +2.5% 도달 → 트레일링 활성화 (peak_pnl 추적 시작)
peak_pnl - current_pnl >= 1.5% → 청산 (TRAILING)
```

| 시나리오 | 결과 |
|---------|------|
| 진입 → +2.5% → +4.0% → +2.5% | peak=4.0%, drawdown=1.5% → **청산 (ROI +7.5%)** |
| 진입 → +2.5% → +8.0% → +6.5% | peak=8.0%, drawdown=1.5% → **청산 (ROI +19.5%)** |
| 진입 → +2.0% → +1.0% → +2.5% → +1.0% | 활성화 안 됨 → SL/TP 또는 시그널 청산 |

**트레일링 활성화 후 하드 TP는 비활성**. 수익이 +10% 이상 달려도 트레일링이 알아서 관리.

### 동작 흐름도

```
가격 변동 (5초마다)
    │
    ├── SL 도달? → YES → 즉시 청산 (STOP LOSS)
    │
    ├── 트레일링 활성? → NO → TP 도달? → YES → 즉시 청산 (TAKE PROFIT)
    │                    │
    │                    └── YES → peak에서 1.5% 하락? → YES → 즉시 청산 (TRAILING)
    │
    └── 수익 >= 2.5%? → YES → 트레일링 활성화 + peak 기록
```

### 현재 구조의 한계

1. **Fixed TP(10%) 사실상 미발동**: 트레일링(2.5%)이 먼저 활성화되므로, 트레일링 없이 10%에 도달하는 경우는 극히 드묾
2. **트레일링 1.5% 고정 trail**: 큰 추세에서도 동일한 폭으로 흔들리면 조기 청산. 변동성 높은 코인(SOL 등)은 노이즈만으로 트레일링에 걸릴 수 있음
3. **2.5% 미만 구간에서 익절 메커니즘 부재**: 수익 +1~2%에서 시그널이 소멸하면 WEAK/FADE 청산에 의존 (최대 25분 지연)

### 로그 파일 구조

```
logs/
├── signals/
│   └── 2026-03-19.jsonl      ← 사이클별 전체 시그널 스냅샷

```

### 시그널 로그 내용 (signals/*.jsonl)

한 행에 포함되는 정보:

| 필드 | 내용 |
|------|------|
| `scores` | 앙상블 최종 스코어 (심볼별) |
| `alphas` | 개별 알파 score/confidence + 내부 메타 (RSI값, z-score 등) |
| `prices` | 시그널 시점 가격 (forward return 사후 계산용) |
| `market` | BTC 가격/수익률, 변동성 |
| `funding` | 최신 펀딩비율 |
| `oi` | OI 변화율 |
| `volumes` | 최근 30분 / 평균 거래량 비율 |
| `ls_ratio` | 롱숏비율 |
| `spreads` | 오더북 스프레드 (bps) |
| `returns` | 코인별 1일/7일 수익률 |
| `decision` | 의사결정 상태 (OPEN/CLOSE/HOLD + 사유) |
| `position` | 현재 포지션 상태 + 미실현 PnL |
| `daily_pnl` | 당일 누적 손익 |
