# Strategy Ensemble — 암호화폐 선물 자동매매 시스템

바이낸스 USDT-M 무기한 선물, 3배 레버리지, 최대 1 포지션 집중 투자.
8개 알파 시그널 → IC 기반 가중합 앙상블 → 규칙 기반 의사결정 → 자동 실행.

```
conda activate strategy_ensemble
python scripts/run_daemon.py          # 실전
python scripts/run_daemon.py --dry-run  # 시그널만 로깅
```

## 3. 8개 알파 — IC 검증 기반 재설계

### 알파 가중치 구성 (2026-03-20 IC 기반 재구성)

모든 알파의 score 범위가 [-1, +1]로 통일됨.

```
핵심 (IC 검증됨, 57%)                    보조 (조건부 유효, 43%)
┌──────────────────────────────────┐    ┌──────────────────────────┐
│ MomentumComposite     20% █████ │    │ IntradayRSIV2       12% │
│ FundingCarryEnhanced  15% ███▓  │    │ MomentumMultiScale  12% │
│ DerivativesSentiment  12% ███   │    │ IntradayVWAPV2      12% │
│ MeanRevMultiHorizon   10% ██▓   │    │ OrderbookImbalance   7% │
└──────────────────────────────────┘    └──────────────────────────┘
                                        SpreadMomentum    0% (비활성)
                                        VolatilityRegime  0% (비활성)
```

### IC (Information Coefficient) 검증 결과

| 알파 | IC(30분) | IC(1시간) | 비고 |
|------|---------|---------|------|
| MeanRevMultiHorizon | +0.086 | **+0.130** | 부호 반전 후 최고 IC |
| DerivativesSentiment | +0.073 | **+0.108** | OI/LS 기반 구조적 예측력 |
| MomentumComposite | +0.064 | **+0.093** | 가장 일관된 양의 IC |
| IntradayRSIV2 | +0.033 | **+0.071** | 부호 반전 후 양전환 |
| FundingCarryEnhanced | +0.033 | **+0.044** | 구조적 edge |
| OrderbookImbalance | +0.046 | **+0.039** | 부호 반전 후 양전환 |
| IntradayVWAPV2 | +0.012 | -0.002 | 극단 이탈 시만 유효 |
| MomentumMultiScale | -0.004 | +0.026 | 단기 예측력 약함 |

### 알파별 상세

#### ① MomentumComposite — 20% (핵심, IC +0.093)

- **데이터**: 일봉 (20일 + 5일)
- **계산**: 절대모멘텀 20일(40%) + 상대모멘텀 순위(30%) + 단기모멘텀 5일(30%) 합산
- **효용**: 20일 추세와 5일 추세를 독립적으로 보고 방향 일치 여부 확인
- **특징**: 3요소 방향 일치 시 confidence=0.50, 20일/5일 불일치 시 0.25
- **스케일**: [-1, +1] (이전 ×0.5 제거)

#### ② FundingCarryEnhanced — 15% (구조적 edge, IC +0.044)

- **데이터**: 펀딩비율 (14일 = 42개 8h 구간) + OI + 일봉
- **계산**: tanh(-평균펀딩 × 2500) → 펀딩속도 가산 보너스(±0.15) → 군중 할인
- **효용**: 펀딩비 음수 = 숏 쪽이 롱에게 지불 → 롱이 구조적으로 유리
- **추세 필터**: 펀딩과 가격이 반대 방향(구조적 불일치) → 신뢰도 1.3배 증가 / 같은 방향(군중 추종) → 0.6배 감소

#### ③ DerivativesSentiment — 12% (스퀴즈 감지, IC +0.108)

- **데이터**: OI 변화율, 펀딩비, 롱숏비율
- **계산**: 스퀴즈 점수(OI 2%+ & 펀딩 불일치) × 60% + 롱숏비율 contrarian × 40%
- **효용**: OI 급증 + 펀딩 역방향 = 스퀴즈 위험 감지. LS > 0.52 = 롱 과잉 → 숏 유리
- **개선**: 기존 OI 5% 임계값 → 2%로 하향, LS 감도 0.55 → 0.52로 확대

#### ④ MeanReversionMultiHorizon — 10% (추세 확인, IC +0.130)

- **데이터**: 일봉 (3/5, 10/20, 20/60일 Z-score 3개)
- **계산**: 3개 시간대 Z-score 평균 → tanh(z/2)
- **핵심 변경**: **부호 반전** — 크립토에서 Z 양수(과매수)는 추세 지속을 의미
- **Z-score 기간 일관성 수정**: SMA/std 기간 매칭 (기존 SMA3/std20 불일치 해소)

#### ⑤ IntradayRSIV2 — 12% (추세추종, IC +0.071)

- **데이터**: 1시간봉 (14개 = 14시간)
- **계산**: Wilder RSI(14) → RSI > 70: 롱 추세 강화, RSI < 30: 숏 추세 강화
- **핵심 변경**: **부호 반전** — 기존(과매수=숏)은 IC 음수. 크립토에서 과매수=추세 지속
- **중립 구간**: RSI 40~60에서 score=0 (침묵)

#### ⑥ MomentumMultiScale — 12% (단기 모멘텀)

- **데이터**: 5분봉 (6/18/36개 = 30분/90분/180분) + 일봉 (변동성 정규화)
- **계산**: 3개 시간대 가중 수익률 (40%/35%/25%) → 일봉 5일 변동성으로 정규화 → tanh
- **스케일**: [-1, +1] (이전 ×0.5 제거)
- **주의**: IC ≈ 0으로 단기 예측력이 약함. 가중치 30% → 12%로 축소

#### ⑦ IntradayVWAPV2 — 12% (평균회귀)

- **데이터**: 1시간봉 (최근 24개 = 1일)
- **계산**: 24시간 VWAP 대비 Z-score → |Z| > 0.8이면 반대 방향 신호
- **효용**: 극단 이탈(Z > 2) 시에만 유효. 78%의 시간 동안 score=0 (침묵)
- **추세 필터**: 4시간 수익률 > 2%이면 신뢰도 50% 감소

#### ⑧ OrderbookImbalance — 7% (contrarian, IC +0.039)

- **데이터**: 실시간 오더북 (3구간: 0-5, 5-10, 10-20 레벨)
- **계산**: 구간별 bid/ask 불균형 가중합 → **부호 반전** (bid 과잉 → 숏)
- **핵심 변경**: IC 분석에서 bid 과잉 후 가격 하락 확인 → contrarian 방식으로 전환
- **필터**: 최근 5 스냅샷 60% 이상 방향 일관 + 가격 모멘텀과 OB가 같은 방향(과열)일 때만

### 카테고리별 효용 정리

| 카테고리 | 알파 | 합산 가중치 | 역할 |
|---------|------|-----------|------|
| **모멘텀/추세** | MomentumComposite, MomentumMultiScale, IntradayRSIV2, MeanRevMulti | 54% | 추세 방향과 강도 |
| **캐리/심리** | FundingCarry, DerivativesSentiment | 27% | 구조적 edge + 스퀴즈 경고 |
| **평균회귀** | IntradayVWAPV2 | 12% | 극단 이탈 시 반전 |
| **Contrarian** | OrderbookImbalance | 7% | 오더북 과열 역추세 |

**설계 의도**: IC 검증으로 실제 수익을 예측하는 알파에 가중치를 집중. 추세추종(54%)이 핵심이고, 캐리(27%)가 구조적 방향을 제공하며, 평균회귀(12%)와 contrarian(7%)이 과열을 견제.

## 4. 앙상블 — 시그널을 하나로 합치는 방법

### 계산 공식

```
                    Σ (score_i × confidence_i × weight_i)
ensemble_score = ─────────────────────────────────────── × vol_confidence
                          Σ weight_i
```

- **score**: 알파가 출력한 방향 신호 (-1 ~ +1, 모든 알파 통일)
- **confidence**: 알파가 자체 판단한 신뢰도 (0 ~ 1)
- **weight**: settings.py의 ALPHA_WEIGHTS
- **vol_confidence**: VolatilityRegime이 출력한 시장 변동성 신뢰도 (최소 0.5로 캡)

### 특징

- **confidence 가중**: 알파가 "이 신호에 확신이 없다"고 판단하면 자동으로 기여도 감소
- **NaN 방어**: score나 confidence가 NaN이면 해당 알파 스킵
- **스케일 통일**: 모든 알파가 [-1, +1] 범위 (기존 MMS/MC의 ×0.5 제거)

### 스코어 해석

| ensemble_score | 해석 |
|---------------|------|
| +0.16 이상 | 롱 진입 후보 |
| -0.18 이하 | 숏 진입 후보 |
| ±0.045 미만 | 시그널 거의 없음 (WEAK) |
| ±0.09 미만 | 시그널 약화 중 (FADE) |

## 5. 진입 로직 — 언제 포지션을 여는가

### 임계값 (비대칭)

| 방향 | 임계값 | 근거 |
|------|--------|------|
| **LONG** | score >= **+0.16** | 스케일 통일(×0.5 제거) 반영. MMS/MC 32% 스케일 2배 → ×1.3 |
| **SHORT** | score <= **-0.18** | 숏은 롱보다 엄격하게 (하락장 리스크 비대칭) |

### 진입 프로세스

```
사이클 1: score +0.18 → PENDING (1/2 사이클)
사이클 2: score +0.19 → 상승 확인 (delta >= 0.01) → OPEN!
```

1. **2사이클 연속 확인**: 1번 넘었다고 바로 진입하지 않음. 2사이클(10분) 연속으로 임계값을 넘어야 함
2. **스코어 상승 요구**: 마지막 스코어 - 첫 스코어 >= 0.01. 시그널이 강해지는 추세여야 진입

### 진입 필터 (차단 조건)

| 필터 | 조건 | 이유 |
|------|------|------|
| **과열 방지** | 해당 코인 1시간 수익률 > ±3% | 이미 많이 움직인 후 추격 매수/매도 방지 |
| **BTC 방향** | BTC 1시간 -1% 이하인데 롱 / +1% 이상인데 숏 | BTC 역방향 진입은 대부분 실패 |
| **거래량 확인** | 최근 30분 거래량 / 최근 3시간 평균 < 0.5 | 거래량 없는 가격 변동은 신뢰 불가 |
| **쿨다운** | 같은 코인 청산 후 20분 | 연속 손절 방지 |
| **연속 손실 차단** | 같은 코인 3회 연속 손실 → 4시간 차단 | 특정 코인에 대한 시그널 반복 실패 시 차단 |
| **일일 횟수** | 하루 최대 6회 거래 | 과도한 매매 방지 |

### 포지션 사이징

- 잔고 × 95% × 3배 레버리지 = 실질 포지션 크기
- 예: 잔고 $1,000 → 포지션 $2,850

## 6. 청산 로직 — 언제 포지션을 닫는가

### 청산 우선순위 (위에서 아래로)

```
우선순위 1: SL/TP 모니터 (5초 체크, 별도 태스크)
    ├── 하드 SL 도달 → 즉시 청산
    ├── 트레일링 활성 중 하락 → 즉시 청산
    └── 하드 TP 도달 (트레일링 미활성시만) → 즉시 청산

우선순위 2: 규칙 엔진 (5분 체크, 메인 루프)
    ├── 최소 보유 15분 미만 → 청산 불가
    ├── REVERSAL: 반대 방향 진입 임계값 도달 → 즉시 청산
    ├── WEAK: |score| < 0.045가 10분 지속 → 청산
    ├── FADE: |score| < 0.09가 15분 지속 → 청산
    └── SWITCH: 30분 이상 보유 + 더 강한 코인 발견 → 청산 후 교체
```

### 규칙 엔진 청산 상세

#### REVERSAL (시그널 반전)

```
롱 보유 중: 현재 score < -0.18 (숏 임계값) → 즉시 청산
숏 보유 중: 현재 score > +0.16 (롱 임계값) → 즉시 청산
```

시장 방향이 완전히 뒤집혔다는 뜻이므로 최소 보유 시간(15분)만 지나면 즉시 실행.

#### WEAK (시그널 소멸)

```
effective_score < 0.045 → 타이머 시작 → 10분 지속 → 청산
(effective_score = score if LONG, -score if SHORT)
```

시그널이 거의 사라진 상태. 10분간 회복하지 않으면 청산.

#### FADE (시그널 약화)

```
effective_score < 0.09 → 타이머 시작 → 15분 지속 → 청산
```

시그널이 진입 임계값(0.16)의 ~56% 수준으로 약해진 상태. 타이머 중 스코어가 회복되면 리셋.

#### SWITCH (코인 교체)

```
조건 (모두 충족):
  - 보유 30분 이상
  - 현재 코인의 effective_score < 0.09 (FADE 수준)
  - 다른 코인의 |score|가 진입 임계값 이상
  - 같은 방향이면 스코어 차이 >= 0.13
  - 반대 방향이면 현재 스코어가 진입 임계값 - 0.10 이하
```

현재 코인보다 훨씬 강한 시그널이 나온 코인이 있을 때, 청산 → 새 코인 진입.

## 7. 익절 로직 — SL/TP/트레일링 상세

### SL/TP 설정 (비대칭)

| 항목 | LONG | SHORT | 3x ROI 기준 |
|------|------|-------|------------|
| **Stop Loss** | -3.5% | -2.5% | 롱 -10.5%, 숏 -7.5% |
| **Take Profit** | +10% | +8% | 롱 +30%, 숏 +24% |
| **RR 비율** | 2.86:1 | 3.2:1 | — |

- 숏의 SL이 더 타이트한 이유: 상방 돌파가 더 빠르고 폭발적이므로 빠르게 손절
- SL/TP 가격은 진입 시점에 계산되어 `PositionStore`에 저장 (봇 재시작해도 유지)

### 트레일링 스탑

```
수익 +2.5% 도달 → 트레일링 활성화 (peak_pnl 추적 시작)
peak - current >= 1.5% → 청산 (TRAILING)
```

| 시나리오 | 결과 |
|---------|------|
| 진입 → +2.5% → +4.0% → +2.5% | peak=4.0%, drawdown=1.5% → **청산 (ROI +7.5%)** |
| 진입 → +2.5% → +8.0% → +6.5% | peak=8.0%, drawdown=1.5% → **청산 (ROI +19.5%)** |
| 진입 → +2.0% → +1.0% → +2.5% → +1.0% | 활성화 안 됨 → SL/TP 또는 시그널 청산 |

**트레일링 활성화 후 하드 TP는 비활성**. 수익이 +10% 이상 달려도 트레일링이 관리.

### 동작 흐름도

```
가격 변동 (5초마다)
    │
    ├── SL 도달? → YES → 즉시 청산 (STOP LOSS)
    │
    ├── 트레일링 활성? → NO → TP 도달? → YES → 즉시 청산 (TAKE PROFIT)
    │                    │
    │                    └── YES → peak에서 1.5% 하락? → YES → 즉시 청산 (TRAILING)
    │
    └── 수익 >= 2.5%? → YES → 트레일링 활성화 + peak 기록
```

## 10. 로그와 모니터링

### 로그 파일 구조

```
logs/
├── signals/
│   └── 2026-03-20.jsonl      ← 5분마다: 전체 시그널 스냅샷
├── trades/
│   └── 2026-03-20.jsonl      ← 거래 시: 진입/청산/사유/PnL/궤적
├── filters/
│   └── 2026-03-20.jsonl      ← 필터 차단 시: 사유/스코어/가격
├── sltp/
│   └── 2026-03-20.jsonl      ← 5초~60초: SL/TP 궤적 (peak/trailing)
└── position_store.json        ← 현재 포지션 (봇 재시작 복구용)
```

### 시그널 로그 (signals/*.jsonl)

5분마다 1행, 행당 ~20KB. 한 행에 포함되는 정보:

| 필드 | 내용 |
|------|------|
| `scores` | 앙상블 최종 스코어 (심볼별) |
| `alphas` | 개별 알파 score/confidence + 내부 메타 (RSI값, z-score 등) |
| `prices` | 시그널 시점 가격 (forward return 사후 계산용) |
| `market` | BTC 가격/수익률, 변동성 |
| `funding` | 최신 펀딩비율 |
| `oi` | OI 변화율 |
| `volumes` | 최근 30분 / 최근 3시간 평균 거래량 비율 |
| `ls_ratio` | 롱숏비율 |
| `spreads` | 오더북 스프레드 (bps) |
| `returns` | 코인별 1일/7일 수익률 |
| `decision` | 의사결정 상태 (OPEN/CLOSE/HOLD + 사유) |
| `position` | 현재 포지션 상태 + 미실현 PnL |
| `daily_pnl` | 당일 누적 손익 |
| `near_threshold` | 임계값 70~100% 구간 코인 (놓친 기회 분석용) |
| `data_quality` | confidence=0 알파 수, 누락 알파 목록 |
| `entry_alphas` | OPEN 시에만: 진입 시점 알파 전체 상세 |
