# Strategy Ensemble 사용설명서

이 문서는 **퀀트 지식이 없는 사람**도 이해할 수 있도록, 이 프로그램이 무엇이고, 어떤 데이터를 쓰고, 어떤 논리로 매매하고, 어떻게 실행하는지를 처음부터 끝까지 설명합니다.

---

## 1. 이 프로그램은 뭘 하는 건가요?

한 줄 요약: **암호화폐 선물 시장에서 자동으로 매매하는 트레이딩 봇**입니다.

좀 더 풀어 쓰면:

1. 바이낸스(Binance)라는 암호화폐 거래소에서 **선물(Futures)** 거래를 합니다
2. 10가지 수학적 분석 방법(**알파**)으로 "지금 뭘 사고팔면 좋을지" 점수를 매깁니다
3. 이 점수들을 AI(Claude Sonnet)에게 보내면, AI가 최종 판단을 내립니다
4. AI의 판단에 따라 자동으로 주문을 넣습니다
5. 이 과정을 **5분마다** 반복합니다

### 알아야 할 기본 용어

| 용어 | 쉬운 설명 |
|------|----------|
| **선물(Futures)** | "나중에 이 가격에 사겠다/팔겠다"라는 계약. 가격이 오를 것 같으면 **롱(Long)**, 내릴 것 같으면 **숏(Short)**을 걸 수 있습니다. 주식과 달리 가격이 떨어져도 돈을 벌 수 있습니다. |
| **레버리지(Leverage)** | 빌린 돈으로 더 크게 베팅하는 것. 이 봇은 **3배 레버리지**를 씁니다. 즉, 100만원이 있으면 300만원어치 거래를 합니다. 수익도 3배, 손실도 3배입니다. |
| **알파(Alpha)** | "시장 평균보다 더 잘할 수 있는 전략". 예를 들어 "최근 3시간 동안 가격이 쭉 올랐으면 앞으로도 오를 가능성이 높다"라는 가설을 수학적으로 계산한 것입니다. |
| **롱(Long)** | "가격이 오를 것이다"에 베팅하는 것. 100원에 롱을 걸고 120원이 되면 20원 이익. |
| **숏(Short)** | "가격이 내릴 것이다"에 베팅하는 것. 100원에 숏을 걸고 80원이 되면 20원 이익. |
| **손절(Stop Loss, SL)** | "여기까지 떨어지면 자동으로 팔겠다"라는 안전장치. 예: 진입가 대비 -5%에 설정. |
| **익절(Take Profit, TP)** | "여기까지 오르면 자동으로 팔겠다"라는 목표가. 예: 진입가 대비 +10%에 설정. |
| **펀딩비율(Funding Rate)** | 선물 시장에서 8시간마다 롱/숏 보유자 간에 주고받는 수수료. 양수면 롱이 숏에게 지불, 음수면 숏이 롱에게 지불. 이걸 이용해 "돈을 받으면서 포지션을 유지하는" 전략이 가능합니다. |
| **USDT** | 달러와 1:1로 연동된 암호화폐. 이 봇의 기본 화폐단위입니다. |

## 2. 어떤 데이터를 사용하나요?

이 봇은 바이낸스 거래소에서 5종류의 데이터를 실시간으로 수집합니다.

### 2-1. 가격 데이터 (OHLCV)

OHLCV는 Open(시가), High(고가), Low(저가), Close(종가), Volume(거래량)의 줄임말입니다. 쉽게 말해 "이 시간 동안 가격이 얼마에서 시작해서, 얼마까지 올랐고, 얼마까지 내렸고, 얼마에서 끝났고, 거래가 얼마나 됐는지"를 기록한 것입니다.

이 데이터를 **세 가지 시간 단위**로 수집합니다:

| 시간 단위 | 수집 범위 | 갱신 주기 | 용도 |
|-----------|----------|----------|------|
| **일봉 (1일)** | 최근 300일 | 4시간마다 | 장기 추세 파악 (20일 모멘텀, 평균회귀 등) |
| **1시간봉** | 최근 7일 | 5분마다 | 중기 시그널 (RSI, VWAP 등) |
| **5분봉** | 최근 5일 | 5분마다 | 단기 타이밍 (인트라데이 모멘텀) |

**왜 여러 시간 단위를 쓰나요?**

주식 차트를 볼 때 "일봉으로 보면 상승 추세인데, 5분봉으로 보면 잠깐 빠지고 있다"라는 상황이 있습니다. 일봉은 "큰 방향"을, 5분봉은 "지금 당장의 움직임"을 보여줍니다. 이 봇은 큰 방향과 작은 움직임을 모두 참고합니다.

### 2-2. 펀딩비율 (Funding Rate)

바이낸스 선물은 8시간마다 "펀딩"이라는 수수료 교환이 일어납니다.

- **펀딩비율이 양수(+)**: 롱 보유자가 숏 보유자에게 돈을 줍니다
- **펀딩비율이 음수(-)**: 숏 보유자가 롱 보유자에게 돈을 줍니다

예를 들어 펀딩비율이 -0.03%이면, 숏을 잡고 있는 사람이 8시간마다 자기 포지션의 0.03%를 롱 보유자에게 지불합니다. 이건 연환산하면 약 -33%에 달하는 비용입니다.

이 봇은 **90일치 펀딩 기록**을 수집해서, "지속적으로 숏이 돈을 내고 있는 코인 → 롱을 잡으면 돈을 받으면서 포지션을 유지할 수 있다"는 식으로 활용합니다.

### 2-3. 오더북 (Orderbook)

오더북은 "지금 이 가격에 얼마만큼의 매수/매도 주문이 대기 중인지"를 보여주는 데이터입니다.

```
매수(Bid) 쪽:          매도(Ask) 쪽:
$99.50  — 500개         $100.50 — 200개
$99.00  — 1,000개       $101.00 — 300개
$98.50  — 800개         $101.50 — 150개
```

만약 매수 쪽에 대기 주문이 훨씬 많다면? → "사려는 사람이 많다 → 가격이 오를 가능성이 높다"고 해석할 수 있습니다. 이것을 **오더북 불균형(imbalance)**이라고 합니다.

이 봇은 상위 10개 코인의 오더북을 **60초마다** 스냅샷하고, 최근 30분의 변화 추이를 추적합니다.

### 2-4. 미결제약정 (Open Interest, OI)

OI는 "현재 청산되지 않은 선물 계약의 총 규모"입니다. 쉽게 말해 "지금 이 코인에 얼마나 많은 사람이 베팅하고 있는지"를 보여줍니다.

- OI가 급격히 증가: 새로운 돈이 시장에 유입 → 큰 움직임 예고
- OI가 감소: 포지션 정리 중 → 변동성 감소 예상

### 2-5. 롱/숏 비율 (Long/Short Ratio)

"전체 포지션 중 롱이 몇 %, 숏이 몇 %인지"를 보여줍니다.

- 롱이 65% 이상: 롱 포지션이 과도하게 쏠림 → 가격 하락 시 대규모 강제 청산(**롱 스퀴즈**) 위험
- 숏이 65% 이상: 숏 포지션이 과도하게 쏠림 → 가격 상승 시 대규모 강제 청산(**숏 스퀴즈**) 위험

### 2-6. 대상 코인 (Universe)

이 봇은 바이낸스 선물 시장에서 **24시간 거래량 상위 20개** 코인을 자동 선별합니다. 단, 다음은 제외합니다:

- 밈 코인 (DOGE, PEPE, SHIB 등) — 극단적 변동성, 분석 불가
- 지수/원자재 파생상품 (금, 은)
- 저유동성 코인

## 3. 10가지 분석 방법 (알파)은 어떻게 작동하나요?

이 봇에는 10개의 알파가 있습니다. 각 알파는 독립적으로 "이 코인을 사야 할까(+점수) 팔아야 할까(-점수)"를 -1 ~ +1 사이의 점수로 출력합니다.

### 알파 목록 한눈에 보기

| # | 이름 | 가중치 | 분류 | 한 줄 설명 |
|---|------|--------|------|----------|
| 1 | MomentumMultiScale | 22% | 모멘텀 | 최근 30분/90분/180분 가격 추세 |
| 2 | FundingCarryEnhanced | 18% | 캐리 | 펀딩비율로 "돈 받으며 버티기" |
| 3 | MomentumComposite | 15% | 모멘텀 | 20일 장기 추세 강도 |
| 4 | IntradayVWAPV2 | 10% | 평균회귀 | 공정가치 대비 이탈도 |
| 5 | IntradayRSIV2 | 8% | 평균회귀 | 과매수/과매도 감지 |
| 6 | DerivativesSentiment | 8% | 캐리 | OI-펀딩 스퀴즈 감지 |
| 7 | MeanReversionMultiHorizon | 8% | 평균회귀 | 다중 시간 축의 되돌림 |
| 8 | OrderbookImbalance | 5% | 마이크로 | 오더북 매수/매도 불균형 |
| 9 | SpreadMomentum | 3% | 마이크로 | 스프레드 변화 + 가격 방향 |
| 10 | VolatilityRegime | 3% | 보조 | 변동성에 따른 신뢰도 조절 |

### 각 알파 상세 설명

#### 1. MomentumMultiScale (22%) — "추세를 따라가자"

**핵심 아이디어**: "최근 올랐으면 더 오를 가능성이 높다" (모멘텀 효과)

**작동 방식**:
1. 5분봉 데이터에서 최근 30분, 90분, 180분의 수익률을 각각 계산
2. 세 개를 가중 평균 (짧은 것과 긴 것을 섞어서 노이즈 감소)
3. **거래량 확인**: 거래량이 평소보다 적으면 → 시그널 약화 (확신 부족)
4. **가속도 체크**: 상승 속도가 빨라지고 있으면 → 시그널 강화

**예시**: ETH가 지난 3시간 동안 +3% 올랐고, 거래량도 평소의 1.5배라면 → 강한 롱(+) 시그널

**왜 가중치가 가장 높나요?**: 암호화폐 시장에서 단기 모멘텀(추세 추종)이 가장 안정적인 수익원입니다. "오르는 건 계속 오르고, 내리는 건 계속 내린다"는 현상이 크립토에서 특히 강합니다.

#### 2. FundingCarryEnhanced (18%) — "수수료를 받으면서 거래하자"

**핵심 아이디어**: "남들이 수수료를 내는 방향의 반대편에 서면, 돈을 받으면서 포지션을 유지할 수 있다"

**작동 방식**:
1. 최근 14일 평균 펀딩비율 계산
2. 펀딩이 지속적으로 음수인 코인 → 롱을 잡으면 8시간마다 수수료를 받음 → 롱 점수 높임
3. **펀딩 속도 체크**: 펀딩이 더 극단적으로 변하고 있으면 → 시그널 강화
4. **군중 할인**: OI가 급증하면서 펀딩도 극단적이면 → "다들 같은 베팅을 하고 있다" → 위험하므로 시그널 약화
5. **추세 확인**: 펀딩이 음수(롱 유리)인데 가격이 이미 +3% 이상 급등했으면 → "이미 늦었다" → 시그널 약화

**예시**: SOL의 펀딩이 14일 연속 음수(-0.02%), OI도 안정적 → 롱을 잡으면 하루에 약 0.06%씩 수수료 수입

#### 3. MomentumComposite (15%) — "장기 추세의 강도"

**핵심 아이디어**: 20일 동안의 가격 움직임을 세 가지 각도로 분석

**세 가지 관점**:
1. **절대 모멘텀**: 이 코인 자체가 20일간 얼마나 올랐/내렸나
2. **상대 모멘텀**: 다른 코인들 대비 이 코인의 순위가 몇 등인가 (전체 20개 중 1등이면 강한 롱)
3. **위험 조정 모멘텀**: 수익률 / 변동성 → "안정적으로 올랐는지, 들쭉날쭉 올랐는지"

세 점수를 40:30:30으로 합산합니다.

**왜 이렇게 복잡하게?**: 단순히 "20% 올랐다"만 보면, 하루에 20% 폭등한 건지 20일 동안 꾸준히 1%씩 오른 건지 구분 못 합니다. 꾸준히 오른 게 더 의미 있으므로 위험 조정이 필요합니다.

#### 4. IntradayVWAPV2 (10%) — "지금 가격이 비싼가 싼가?"

**핵심 아이디어**: VWAP(거래량 가중 평균 가격)은 "오늘 하루 동안 거래된 평균 가격"입니다. 현재 가격이 이 평균보다 많이 위에 있으면 "비싸다 → 내릴 수 있다", 아래에 있으면 "싸다 → 오를 수 있다".

**작동 방식**:
1. 최근 24시간의 1시간봉으로 VWAP 계산
2. 현재 가격과 VWAP의 차이를 표준편차로 나눔 (정규화)
3. 1.5 표준편차 이상 벗어났을 때만 시그널 발생 (안에 있으면 거의 무시)
4. 강한 추세 중이면 시그널 약화 (상승장에서 VWAP 위에 있는 건 정상)

**비유**: 마트에서 평소 1,000원에 팔던 물건이 갑자기 1,500원이면 "비싸다, 곧 내릴 거다"라고 생각하는 것과 같습니다.

#### 5. IntradayRSIV2 (8%) — "과열인가 침체인가?"

**핵심 아이디어**: RSI(Relative Strength Index)는 "최근 상승 폭 vs 하락 폭"을 비교하는 지표. 0~100 사이 값을 가집니다.

- RSI > 70: **과매수** → "너무 많이 올랐다, 쉬어갈 때다" → 매도(-) 시그널
- RSI < 30: **과매도** → "너무 많이 떨어졌다, 반등할 때다" → 매수(+) 시그널
- RSI 30~70: 중립 (약한 시그널만)

**작동 방식**: 1시간봉 14기간 RSI 계산 → 과매수/과매도 영역에서만 강한 시그널

#### 6. DerivativesSentiment (8%) — "스퀴즈가 올 것 같다"

**핵심 아이디어**: OI(미결제약정)와 펀딩비율의 **불일치**를 감지해서 "대규모 강제 청산(스퀴즈)"을 예측

**스퀴즈란?**: 한 방향에 베팅한 사람이 너무 많으면, 반대로 가격이 움직일 때 연쇄적으로 강제 청산되면서 폭등/폭락이 일어나는 현상

**감지 로직**:
- OI 증가 + 펀딩 음수(숏이 돈을 냄) → 숏이 쌓이고 있다 → 가격 오르면 숏 스퀴즈 → **롱 시그널**
- OI 증가 + 펀딩 양수(롱이 돈을 냄) → 롱이 쌓이고 있다 → 가격 내리면 롱 스퀴즈 → **숏 시그널**

#### 7. MeanReversionMultiHorizon (8%) — "너무 많이 갔으면 돌아온다"

**핵심 아이디어**: 가격이 평균에서 너무 멀리 벗어나면, 결국 평균으로 돌아오는 경향(평균회귀)을 이용

**작동 방식**:
1. 세 가지 시간 축에서 Z-score 계산:
   - 3일/20일 (단기 과열)
   - 5일/60일 (중기 과열)
   - 10일/120일 (장기 과열)
2. 세 Z-score가 모두 "과매수"를 가리키면 → 신뢰도 높은 매도 시그널
3. 볼린저 밴드로 추가 확인

**Z-score란?**: "지금 값이 평균에서 표준편차 몇 개만큼 떨어져 있는지"를 나타내는 수치. Z-score가 +2면 "평소보다 2표준편차나 높다 → 매우 이례적 → 되돌릴 가능성 높다".

#### 8. OrderbookImbalance (5%) — "사려는 사람이 더 많다"

**핵심 아이디어**: 오더북에서 매수 대기량이 매도보다 훨씬 많으면 → 가격 상승 압력

**작동 방식**:
1. 오더북의 상위 20단계를 세 구간으로 나눔 (1-5, 5-10, 10-20)
2. 각 구간에서 매수 금액 vs 매도 금액의 불균형 계산
3. 가까운 단계(1-5)에 50%, 중간(5-10)에 30%, 먼 단계(10-20)에 20% 가중치

**왜 가중치가 낮나요?**: 오더북은 변하기 쉽고(가짜 주문이 많음), 히스토리컬 백테스트가 불가능해서 아직 검증되지 않았습니다. 실전 IC가 확인되면 점진적으로 올릴 예정입니다.

#### 9. SpreadMomentum (3%) — "스프레드가 좁아지고 있다"

**핵심 아이디어**: 매수-매도 호가 차이(스프레드)가 줄어들면서 가격이 오르면 → "확신을 가진 상승" → 지속 가능성 높음

반대로, 스프레드가 벌어지면서 가격이 오르면 → "불안한 상승" → 꺾일 수 있음

#### 10. VolatilityRegime (3%) — "지금 시장이 얼마나 불안한가?"

**핵심 아이디어**: 이 알파는 방향을 예측하지 않습니다. 대신 "지금 변동성이 극단적인가?"를 측정해서 다른 모든 알파의 **신뢰도를 조절**합니다.

- 변동성 정상: 신뢰도 100% (다른 알파 시그널 그대로 사용)
- 변동성 극단적으로 높음: 신뢰도 40~70%로 낮춤 (모든 시그널 축소)

**왜 필요한가요?**: 시장이 매우 불안할 때는 어떤 분석도 틀릴 가능성이 높습니다. 이때 베팅 크기를 줄이는 것이 합리적입니다.

## 4. 알파 점수를 어떻게 합치나요? (앙상블)

10개 알파가 각각 점수를 내면, 이것을 하나의 최종 점수로 합쳐야 합니다. 이 과정을 **앙상블(Ensemble)**이라고 합니다.

### 방법 1: 가중합 (Fallback — 운영 초기 60일)

가장 단순한 방법입니다:

```
최종 점수 = (알파1 점수 x 22%) + (알파2 점수 x 18%) + ... + (알파10 점수 x 3%)
```

여기에 세 가지 보정이 들어갑니다:

1. **신뢰도 가중**: 각 알파가 "이번 시그널은 확신이 높다/낮다"라고 자체 평가한 값을 곱합니다
2. **카테고리 내 상관 감쇄**: 같은 카테고리(예: 모멘텀) 내에서 여러 알파가 같은 방향을 가리키면, 겹치는 정보이므로 약간 깎습니다
3. **변동성 조절**: VolatilityRegime의 신뢰도를 최종 점수에 곱합니다

### 방법 2: Stacking Meta-Learning (운영 60일 이후 자동 전환)

가중합의 한계: 가중치가 고정이고, 알파 간 **상호작용**을 못 잡습니다.

예를 들어:
- RSI가 과매도(30 이하)이면서 + 펀딩이 음수이면 → 시너지 효과 (단순 합산보다 훨씬 강한 시그널)
- 모멘텀이 강한 롱이면서 + 평균회귀가 강한 숏이면 → 상충 (둘 다 확신이 높지만 방향이 반대)

가중합은 이런 것을 구분 못 합니다. Stacking은 이걸 **학습**합니다.

**Stacking 작동 원리**:

```
학습 데이터 (매일 x 20개 코인 = 하루 20행):
  X = [알파1 점수, 알파2 점수, ..., 알파10 점수, 시장 레짐, 변동성]
  y = 5일 후 실제 수익률

모델: Ridge 회귀 (안정적, 과적합 적음)
  → "어떤 알파 조합이 실제로 돈이 됐는지" 과거 데이터에서 학습
  → 알파 간 상호작용 자동 포착
```

**학습 사이클**:
- 최근 90일 데이터로 학습
- 30일마다 자동 재학습 (시장 환경 변화에 적응)
- 학습 데이터 부족 시(< 60일) 가중합으로 자동 fallback

## 5. AI(Sonnet)는 어떤 역할을 하나요?

알파들이 점수를 내고 앙상블이 합산한 후, 최종 판단은 **Claude Sonnet**(AI)이 합니다.

Sonnet은 **단순 실행자가 아니라 최종 의사결정자**입니다. 알파가 "롱을 잡아라"고 해도 Sonnet이 "위험하다"고 판단하면 안 잡습니다.

### Sonnet이 받는 정보 (매 5분마다)

```
1. 최근 트레이딩 성적: 24시간/7일 승률, 순이익
2. 현재 포지션: 어떤 코인을, 어떤 방향으로, 언제부터 잡고 있는지
3. 시장 상황: BTC/ETH 가격, 24시간/7일/30일 수익률, 시장 레짐
4. 파생상품 센티먼트: 코인별 OI 변화, 롱/숏 비율
5. 알파 시그널 합산 점수 + 개별 알파 기여도 분해
6. 스퀴즈 경고: 위험한 상황이 감지되면 경고 메시지
7. 펀딩비율: 주요 코인의 8시간 펀딩 비용
8. 계좌 잔고
```

### Sonnet이 출력하는 것

```json
{
  "positions": [
    {
      "ticker": "ETH/USDT:USDT",
      "action": "LONG",
      "weight": 0.95,
      "stop_loss_pct": -0.05,
      "take_profit_pct": 0.10,
      "reasoning": "강한 모멘텀 + 펀딩 음수, 캐리 수입 가능"
    }
  ],
  "market_assessment": "BTC 저항선 돌파, 시장 전반 불리시",
  "risk_note": "ETH OI 급증 중, 스퀴즈 주의"
}
```

### Sonnet의 안전장치

- 포지션은 최대 **1개**만 유지
- 최소 **30분** 이상 보유 후에만 청산 가능
- 같은 코인을 청산 후 **30분** 동안 재진입 불가
- 손절: 진입가 대비 **-2% ~ -8%** 범위
- 익절: 진입가 대비 **+3% ~ +15%** 범위

## 6. 손절/익절/트레일링 스탑은 어떻게 작동하나요?

포지션을 잡은 후에는 **5초마다** 가격을 확인해서 자동으로 보호합니다.

### 6-1. 하드 손절 (Hard Stop Loss)

"이 가격 이하로 떨어지면 무조건 즉시 청산"

예시: ETH를 $3,000에 롱, 손절 -5% → $2,850 이하로 떨어지면 자동 청산

### 6-2. 트레일링 스탑 (Trailing Stop)

"이익이 충분히 나면, 최고점에서 일정 비율 떨어질 때 청산"

작동 순서:
1. 포지션 수익이 **+2.5%** 도달 → 트레일링 스탑 활성화
2. 이후 가격이 더 올라가면 최고점을 계속 갱신
3. 최고점에서 **-1.5%** 하락하면 → 자동 청산

**예시**:
```
ETH 롱 진입: $3,000
수익 +2.5% ($3,075) 도달 → 트레일링 활성화
가격 계속 상승 → $3,200 (최고점 갱신)
$3,200에서 -1.5% = $3,152 이하로 떨어지면 → 청산
결과: $3,000 → $3,152 = +5.1% 수익 (레버리지 3배 → 약 +15.3% ROI)
```

### 6-3. 고정 익절 (Fixed Take Profit)

트레일링 스탑이 아직 활성화되지 않은 상태에서, 미리 정한 목표가에 도달하면 청산.

## 7. 백테스팅은 어떻게 하나요?

**백테스트(Backtest)**란 "과거 데이터에서 이 전략이 얼마나 잘 됐을까?"를 시뮬레이션하는 것입니다.

### 7-1. 무엇을 측정하나요?

| 지표 | 쉬운 설명 | 좋은 값 |
|------|----------|---------|
| **IC** | "알파 점수가 높은 코인이 실제로 많이 올랐는가?" 점수와 수익률의 상관관계. | > +0.03 |
| **IR** | IC의 안정성. IC가 매일 비슷하면 높고, 들쭉날쭉이면 낮음. | > 0.3 |
| **Hit Rate** | "알파가 '오를 것'이라고 한 코인이 실제로 올랐는가?"의 비율. | > 52% |
| **Turnover** | 시그널이 얼마나 자주 바뀌는지. 너무 높으면 거래 비용 증가. | 0.1~0.5 |
| **IC Decay** | 시그널이 며칠 후까지 유효한지. 1일 IC vs 20일 IC를 비교. | 서서히 감소 |
| **레짐별 IC** | 상승장/하락장/횡보장에서 각각 IC가 어떤지. | 균등 |

### 7-2. 백테스트 방법 (Look-ahead bias 방지)

**Look-ahead bias**란 "미래 데이터를 몰래 보고 전략을 짜는 실수"입니다. 이 봇은 이를 엄격히 방지합니다:

```
날짜 t에서 알파 평가:
  1. t 이전 데이터만 사용 → 시그널 생성 (미래 데이터 절대 안 봄)
  2. 각 코인의 score 기록
  3. t+1일, t+5일, t+10일, t+20일 후의 실제 수익률과 비교
  4. Spearman 순위 상관계수 = IC
```

이 과정을 60일간 반복해서 평균을 냅니다.

### 7-3. 백테스트 가능한 알파 vs 불가능한 알파

| 백테스트 가능 (일봉 데이터) | 백테스트 불가 (실시간 데이터 필요) |
|---------------------------|-------------------------------|
| MomentumComposite | MomentumMultiScale (5분봉) |
| FundingCarryEnhanced | IntradayVWAPV2 (1시간봉) |
| MeanReversionMultiHorizon | IntradayRSIV2 (1시간봉) |
| | OrderbookImbalance (오더북) |
| | SpreadMomentum (오더북) |
| | DerivativesSentiment (OI) |
| | VolatilityRegime (비방향성) |

백테스트 불가능한 알파들은 **라이브 운영 중에 자동으로 IC를 추적**하고, 30일 trailing IC가 지속적으로 나쁘면 가중치를 자동으로 줄입니다.

### 7-4. 백테스트 실행 방법

```bash
# v2 알파 평가 (일봉 기반 3개)
python scripts/evaluate_alphas.py --version v2

# v1 vs v2 비교
python scripts/evaluate_alphas.py --version all

# 90일 평가 + 상관행렬 + CSV 저장
python scripts/evaluate_alphas.py --version all --days 90 --correlation --output results.csv
```

출력 예시:
```
──────────────────────────────────────────────────
  MomentumComposite
──────────────────────────────────────────────────
        IC (1일): +0.0312
        IC (5일): +0.0587
       IR (5일): +0.412
  Hit Rate (5일): 54.2%
       Turnover: 0.234
        평가일 수: 60
```

## 9. 기대 효과와 리스크

### 기대되는 효과

1. **감정 배제**: 사람은 공포/탐욕에 흔들리지만, 봇은 정해진 규칙을 따릅니다
2. **24시간 운영**: 암호화폐 시장은 쉬지 않으므로, 봇도 쉬지 않습니다
3. **다각화된 시그널**: 10개 알파가 서로 다른 관점에서 시장을 분석합니다
4. **자동 위험 관리**: 손절/트레일링 스탑이 5초마다 작동합니다
5. **시장 적응**: Stacking 모델이 30일마다 재학습하여 환경 변화에 적응합니다
6. **캐리 수입**: 펀딩비율을 활용하여 포지션 유지만으로도 수수료 수입 가능

### 리스크

1. **레버리지 리스크**: 3배 레버리지는 수익도 3배이지만 손실도 3배입니다
2. **모델 리스크**: 과거에 잘 작동한 전략이 미래에도 작동한다는 보장은 없습니다
3. **유동성 리스크**: 극단적 시장 상황에서는 손절 주문이 원하는 가격에 체결되지 않을 수 있습니다
4. **기술 리스크**: 서버 다운, 네트워크 장애, 거래소 점검 시 봇이 작동하지 않을 수 있습니다
5. **API 비용**: Claude Sonnet API 호출에 비용이 발생합니다 (5분마다 1회)

## 11. 자주 묻는 질문

**Q: 봇을 끄면 포지션은 어떻게 되나요?**
A: 바이낸스에 열린 포지션은 그대로 남아 있습니다. 봇을 다시 키면 기존 포지션을 인식하고 관리를 재개합니다. 단, 봇이 꺼진 동안에는 손절/익절이 작동하지 않으므로 주의하세요.

**Q: 얼마의 자본이 필요한가요?**
A: 바이낸스 선물 최소 주문이 $5이므로, 3배 레버리지 기준 약 $2 이상이면 기술적으로 가능합니다. 하지만 거래 수수료와 변동성을 고려하면 최소 $100~$500 이상을 권장합니다.

**Q: API 비용은 얼마인가요?**
A: Claude Sonnet API 비용은 5분마다 1회 호출 x 약 $0.01~0.03/회 = 하루 약 $3~9 수준입니다. 바이낸스 API는 무료입니다.

**Q: 60일 전에도 Stacking을 쓸 수 있나요?**
A: 가능하지만 권장하지 않습니다. 학습 데이터가 부족하면 과적합(과거에만 맞는 모델)이 됩니다. 60일은 최소한의 안전 기간입니다. 그 전에는 가중합이 자동으로 적용됩니다.

**Q: 어떤 코인을 거래하나요?**
A: 프로그램이 자동으로 거래량 상위 20개 코인을 선별합니다. 밈 코인, 지수 파생상품 등은 자동 제외됩니다. 특정 코인을 거래하거나 제외하고 싶으면 `config/settings.py`의 `UNIVERSE` 설정을 수정하세요.
